# User Priors — Reference for the Bias Agent (Step 7)

**This is a template.** Copy it to `user-priors.md`, delete the example content, and write your own. The pipeline's Bias Agent checks every draft against this file — it is only as good as this document is honest. A flattering priors file produces a flattering (useless) bias check.

**Writing rules:**
- Write your leanings as an honest observer would describe them, not as you'd defend them.
- For every leaning, name the specific analytical failure it produces when it outruns the evidence.
- This file is private by default. Its value comes from a candor you probably wouldn't publish. (If you share your pipeline, share this template — not your filled-in copy.)
- Standing rule for the Bias Agent: **alignment is not bias.** A conclusion that matches these priors AND is independently carried by the evidence chain passes. The flag is for conclusions where the prior is doing work the evidence isn't.

---

*Everything below is a worked example for a hypothetical centrist/institutionalist. It exists to show the level of candor required. Replace all of it with your own.*

## Values and leanings

- **Institutionalist by default.** Trusts that courts, agencies, and established processes mostly work, and that official findings are mostly reliable. Default skepticism toward claims that a major institution is captured, corrupt, or acting in bad faith. Watch for treating institutional output as ground truth when the institution's own record is part of what's being tested.
- **Incrementalist.** Prefers reform over rupture and assumes durable change happens gradually. Watch for grading urgent evidence on a curve because the implied remedy feels destabilizing.
- **Allergic to "extreme" conclusions.** Instinctively discounts claims that sound radical, regardless of the evidence behind them. Watch for confusing the *tone* of a claim with its *probability* — some true things sound extreme.

## Analytical habits to check

- **Midpoint bias.** Prone to assuming the truth sits between the two loudest positions. It often doesn't; sometimes one side is simply wrong on the facts. Check whether the draft split a difference the evidence didn't split.
- **False balance.** Tempted to pair every criticism with an equivalent from the other side, even when the record is asymmetric. Check whether "both sides" language is describing the evidence or performing fairness.
- **Deference to official sources.** Inclined to weight an agency's numbers over outside analysis by default. Primary does not mean trustworthy; it means primary. Check whether official figures were stress-tested like everything else.
- **Intent avoidance.** Reluctant to attribute intent to institutions or leaders even when direct evidence exists (statements, internal documents, sworn testimony). The tolerance-vs-intent rule cuts both ways: don't claim intent without evidence, and don't refuse it when the evidence is there.

## Counterexamples — when the prior was right

The Bias Agent checks both directions: this section catches **over-correction** — suppressing evidence-supported claims because they match a prior. Each entry is a case where a listed prior was vindicated by the evidence chain. Add entries whenever a Ref Agent ruling shows a prior was carried by the evidence.

- **2020 election fraud claims (example entry).** The institutionalist prior — that courts and audits function — was vindicated: dozens of court losses, recounts, and state audits (many run by officials of the claiming party) converged on the same result. Trusting that process was alignment with the evidence, not deference. Lesson: the prior isn't wrong; it's wrong *by default*. Test it every time, and record when it passes.

## What this file is NOT

- Not a mandate to inject balance, or its opposite. The pipeline rejects neutrality theater and overclaiming alike.
- Not a reason to soften established findings. If the evidence convicts, say so plainly.
- Not static. Edit this file as self-knowledge improves or leanings shift. The Counterexamples section grows the same way — the Ref Agent adds an entry whenever an over-correction ruling shows a prior was carried by the evidence.
